# CLAUDE.md — Задание: проверка всех интеграций перед хакатоном

Ты выполняешь предхакатонную валидацию. Завтра ETHGlobal NYC, мы строим social prediction market (фид + бинарные YES/NO рынки) со стеком интеграций. Твоя задача СЕГОДНЯ — пройтись по каждой интеграции, собрать минимальный рабочий пример и вынести вердикт: работает ли она так, как мы ожидаем. Не продукт, не качество — только вердикты и грабли.

## Глобальные правила

1. Рабочая папка `~/hack-sandbox/`, по подпапке на интеграцию: `arc/ cre/ dynamic/ blink/ worldid/ ens/ uniswap/`. Код одноразовый.
2. **Arc, Chainlink CRE, Dynamic Flow, Blink — новые продукты, их НЕТ в твоих знаниях.** По ним запрещено писать код по памяти. Алгоритм: скачай официальные доки → сохрани в `~/hack-docs/<имя>/` → читай → пиши. Сначала проверяй `<docs-домен>/llms.txt` и `/llms-full.txt`. Если доки не отвечают на вопрос — остановись и спроси меня, не выдумывай методы API.
3. По каждой интеграции работай ровно до вердикта PASS/FAIL по критерию ниже. Никаких рефакторингов, абстракций, «дополнительно я ещё сделал».
4. Таймбоксы жёсткие. Истёк — фиксируй FAIL/PARTIAL с описанием, где застрял, и переходи к следующей.
5. Секреты только в `.env` (в `.gitignore`), проси их у меня перед стартом каждого блока. Не вставляй ключи в код и не логируй их.
6. После каждой интеграции — строка в `~/hack-sandbox/RESULTS.md`:
   `| модуль | PASS/PARTIAL/FAIL | минут потрачено | ожидание совпало? | грабли/заметки |`
7. Тесты, где нужен браузерный клик (логин, виджеты), доводи до состояния «запущено, открой localhost:3000» и зови меня — вердикт по UI ставлю я, ты его записываешь. Не репорти PASS по UI, которого не видел.
8. Порядок выполнения строго: 1 → 2 → 3 → 4 → 5 → 6 → 7. Первые два — критические, от них зависит план.

## Тест 1. Arc testnet (КРИТИЧЕСКИЙ, таймбокс 1.5 ч)

**Что ожидаем:** Arc — EVM L1 от Circle; стандартный Foundry-флоу работает; есть testnet RPC, faucet, эксплорер; USDC — нативный газ/основной актив; обычный Solidity деплоится без модификаций.

**Нужно от меня:** приватник тестового кошелька, RPC URL и faucet (если сам не найдёшь в доках docs.arc.io).

**Шаги:**
1. Скачай доки Arc (getting started, network config, USDC addresses) в `~/hack-docs/arc/`.
2. Выпиши в `arc/NOTES.md`: chain id, RPC, чем платится газ, адрес USDC, эксплорер, как верифицировать контракт.
3. `forge init`, задеплой Counter.sol через `forge create` (или способ из доков, если стандартный не работает).
4. Вызови инкремент, проверь чтение состояния.
5. Сделай transfer тестового USDC на второй адрес, получи tx hash.

**PASS:** контракт задеплоен + tx виден в эксплорере + USDC-перевод прошёл. 
**FAIL:** запиши, что именно сломано (RPC? gas-модель? faucet пустой?) — это решает, переезжаем ли на Base Sepolia.

## Тест 2. Chainlink CRE (КРИТИЧЕСКИЙ, таймбокс 2 ч — не превышать)

**Что ожидаем:** CRE CLI ставится локально; шаблон из официального repo симулируется командой CLI без деплоя; workflow умеет HTTP-fetch внешнего API и EVM-write; для приза достаточно успешной симуляции.

**Шаги:**
1. Скачай доки docs.chain.link/cre (SDK TS + CLI reference) в `~/hack-docs/cre/`.
2. Установи CRE CLI (способ — из доков).
3. Склонируй github.com/smartcontractkit/cre-templates, выбери шаблон, ближайший к «HTTP fetch → EVM write».
4. Запусти симуляцию шаблона БЕЗ изменений (команду возьми из README шаблона).
5. Если шаг 4 прошёл — минимальная модификация: fetch цены ETH с Coingecko `/simple/price`. Снова симуляция.

**PASS:** симуляция отрабатывает (хотя бы немодифицированный шаблон). 
**PARTIAL:** шаблон симулируется, модификация — нет. 
**FAIL:** CLI/симуляция не завелись за 2 часа → в `RESULTS.md` подробно: на каком шаге, какая ошибка. Это вырежет модуль на $6-8K из плана, поэтому опиши затык максимально конкретно.
